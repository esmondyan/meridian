# OSRS 倒卖助手 — 产品设计框架（草稿）

---

## 一、目标客户画像

### 1. 休闲倒卖者
- **特征：** 每天玩 1-2h，身家 1M~10M GP，同时打怪/练技能时顺便倒卖
- **典型场景：** "我身上只有 500 万，有没有今天能倒的东西？"
- **最大痛点：** 不知道怎么开始，4000件物品无从下手

### 2. 专职倒卖者
- **特征：** 每天 3-6h 在线交易，10M~200M+ GP 资金，把倒卖当游戏内的主业
- **典型场景：** "我每天花几个小时点价，能不能告诉我哪些单子最值得去抢？"
- **最大痛点：** 人工盯盘效率低，错过好价，需要别人帮筛选

### 3. 金农 / 委内瑞拉玩家（价格敏感型）
- **特征：** 以 OSRS 为收入来源，走 G2G 换汇，追求稳定低风险收益
- **典型场景：** "我需要每天稳定赚几百万，但不能冒亏本的风险。"
- **最大痛点：** 本金有限、经不起亏损，需要稳健策略，对美元汇率敏感

### 4. 高端庄家 / 商团成员
- **特征：** 500M+ 资金，有组织操作市场，关注稀缺品/限量品
- **典型场景：** "我发现古城新出的装备有套利空间，想监控价格走势。"
- **最大痛点：** 依赖手动盯盘或 Excel，缺自动化的趋势预警工具

## 二、客户要解决的核心需求（按优先级）

### 🥇 第一层：**「我今天倒什么？」**（高频刚需）
> 每天上线打开工具，3秒钟内就知道今天有哪些值得倒卖的单子。

- ✅ 已经解决：评分算法 + 早报推送
- ❌ 未解决：早报推荐的是否是**真正值得倒的**？（今天的 Harmonised orb 亏钱还上榜）
- ❌ 未解决：没有按**用户本金**做个性化推荐

### 🥇 第二层：**「这个单子到底能赚多少？」**（信任感）
> 用户需要知道税后净利润、资本回报率、多久能卖出去。

- ✅ 部分解决：税后利润已计算
- ❌ 未解决：缺少 RoC（资本回报率）— 2.8 亿赚 35 万 vs 17 万赚 1.3 万，用户需要知道哪个效率更高
- ❌ 未解决：缺少**估算卖出时间** — 交易量高≠成交快，需要更精准的流动性指标

### 🥈 第三层：**「怎么不让我错过好价？」**（实时性）
> 用户不想每 5 分钟刷新一次页面，也不想起床后发现昨天的好价没了。

- ❌ 未解决：Discord/Telegram Bot 推送尚未实现（只做了早报，没有实时价格触发）
- ❌ 未解决：没有价格异常告警（突然的价差扩大/缩小）

### 🥈 第四层：**「帮我算账」**（决策辅助）
> 投入 X 万金币，每天能赚多少？实际换算成美元是多少？

- ❌ 未解决：缺"预算模拟器2.0" — 输入你的本金，自动推荐最优配置
- ❌ 未解决：缺批量倒卖计划 — 分散资金到多个物品的最佳组合
- ✅ 已初步实现：预算模拟器（基础版本）

### 🥉 第五层：**「市场在变什么？」**（洞察力）
> 长期追踪哪些品类在涨/在跌，发现隐藏的趋势。

- ❌ 未解决：缺价格走势图的移动端便捷查看
- ❌ 未解决：缺品类级趋势分析（rune 市场整体走低？装备在涨？）
- ✅ 已初步实现：单品价格趋势（Web 端）

## 四、产品设计框架（建议）

### 4.1 核心价值主张
> **「GE Tracker 给你数据。我们教你怎么操作。」**

### 4.2 信息架构

```
┌─ Dashboard ──────────────────────────┐
│  今日概览（用户最需要马上看到的东西）        │
│  ┌─ 我的资金: [输入框]                 │
│  ├─ 今日推荐 TOP5（个性化）             │
│  └─ 市场速览: 交易最活跃 / 价差最大       │
├──────────────────────────────────────┤
│  📋 我要倒卖                          │
│  ├─ 热门推荐（RoC排序，默认）            │
│  ├─ 按资金筛选（输入你的金币量）          │
│  ├─ 自定义筛选（品类/价格/天数）           │
│  └─ 评分排行（A+/A/B/C/D）             │
├──────────────────────────────────────┤
│  📈 市场洞察                          │
│  ├─ 价格趋势（单品走势）                 │
│  ├─ 品类风向（rune/武器/护甲/材料等涨跌） │
│  └─ 热门品类实时成交                    │
├──────────────────────────────────────┤
│  🔔 我的通知                          │
│  ├─ 价格触达（设定目标价）               │
│  ├─ 倒卖机会（"注意！价差突然扩大2倍"）   │
│  └─ 早报回顾                           │
└──────────────────────────────────────┘
```

### 4.3 每一项要做的事情

#### P0（必须做 — 产品成立的基础）
- **✅ 核心修复** — 评分算法加入 RoC 维度 + 排除负利润物品
- **✅ 个性化推荐** — 输入你的资金 → 只推荐你买得起的 + RoC 最高的
- **❌ 英语化** — 对标欧美用户，UI 全英文，Dashboard tab 名和列标题都要英文

#### P1（重要 — 用户会为此付费）
- **❌ Discord Bot 推送** — 价格触达、倒卖机会实时通知
- **❌ Telegram Bot 推送** — 同上，多覆盖一个平台
- **❌"利润计算器"** — 输入买入价/卖出价/数量 → 自动算税后利润 + RoC

#### P2（加分项 — 拉开差距）
- **❌ 品类趋势分析** — 哪个品类整体在涨/跌
- **❌ 倒卖组合优化** — "我有 5000 万，怎么分到不同物品上收益最大化？"
- **❌ 移动端 PWA** — 手机浏览器能当 App 用

#### P3（锦上添花）
- **❌ 历史数据回放** — 查看过去某天的倒卖机会
- **❌ 社区评分** — 用户可以对推荐物品打反馈，"这个价实际挂上去没人买"

## 五、接下来要讨论的问题

1. **RoC 加入评分算法后，A+ 标准要重新校准吗？**（今天的数据看，RoC>5% 才 10 件物品，如果按这个标准 A+ 会变很少，可能 A+ 阈值需要下调）
2. **按资金推荐怎么交互？** 输入框直接筛选，还是做个"你的资金量：新手/进阶/老手"三档？
3. **推送协议：** Discord Bot 还是 Telegram Bot 优先做？（从用户 base 来看 Discord 覆盖面更广）
4. **用户付费模式设想？** 免费版看基本推荐，Pro 版才有实时推送 + 品类趋势？还是先免费养用户再考虑变现？
5. **早报格式改版：** 从"按评分排序"改成"按 RoC 排序 + 标出你买得起"？
